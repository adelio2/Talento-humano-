## Escenarios 

| Código | Caso de Uso | Escenario |
|---|---|---|
| ECU-01 | Registrar solicitud de permiso laboral | El empleado ingresa al sistema, accede al módulo de solicitudes y diligencia el formulario de permiso laboral. El sistema valida los campos obligatorios, registra la solicitud, asigna un número de radicado y deja el estado en "Pendiente". |
| ECU-02 | Solicitar vacaciones | El empleado selecciona la opción de vacaciones, registra las fechas solicitadas y envía la solicitud. El sistema verifica la información registrada, almacena la solicitud y notifica al área de Talento Humano para su revisión. |
| ECU-03 | Registrar incapacidad médica | El empleado registra una incapacidad médica y adjunta el documento de soporte correspondiente. El sistema valida el archivo adjunto antes de permitir el envío de la solicitud. |
| ECU-04 | Solicitar certificación laboral | El empleado realiza una solicitud de certificación laboral desde la plataforma institucional. El sistema registra la solicitud y genera confirmación automática de recepción. |
| ECU-05 | Consultar estado de solicitud | El empleado accede al historial de solicitudes y consulta el estado actualizado de una solicitud específica. El sistema muestra información como fecha de registro, responsable asignado y estado actual. |
| ECU-06 | Consultar historial de solicitudes | El empleado visualiza todas las solicitudes realizadas anteriormente junto con sus fechas, respuestas y estados registrados. |
| ECU-07 | Consultar responsable asignado | El empleado consulta el detalle de una solicitud y el sistema muestra el nombre o cargo del responsable encargado de gestionarla. |
| ECU-08 | Recibir notificaciones automáticas | Cuando una solicitud cambia de estado, el sistema envía automáticamente una notificación al empleado indicando si fue aprobada, rechazada o puesta en revisión. |
| ECU-09 | Visualizar solicitudes registradas | El responsable de Talento Humano accede al panel administrativo y consulta todas las solicitudes registradas en el sistema. |
| ECU-10 | Aprobar solicitud | El responsable de Talento Humano revisa la solicitud registrada y cambia su estado a "Aprobada". El sistema registra la acción realizada y notifica al empleado. |
| ECU-11 | Rechazar solicitud | El responsable de Talento Humano rechaza una solicitud ingresando una observación obligatoria con el motivo del rechazo. |
| ECU-12 | Cambiar estado de solicitud | El responsable de Talento Humano modifica el estado de una solicitud a "En revisión", "Aprobada" o "Rechazada". El sistema almacena el cambio en el historial. |
| ECU-13 | Consultar historial y trazabilidad | El responsable de Talento Humano consulta el historial completo de acciones realizadas sobre una solicitud para verificar fechas, responsables y cambios de estado. |
| ECU-14 | Generar reportes administrativos | El responsable de Talento Humano genera reportes relacionados con cantidad de solicitudes, tiempos de respuesta y estados de aprobación. |
| ECU-15 | Gestionar responsables del proceso | El sistema permite asignar responsables específicos para gestionar solicitudes según el tipo de proceso o solicitud registrada. |
| ECU-16 | Validar información de solicitudes | Antes de registrar una solicitud, el sistema valida que todos los campos obligatorios estén completos y que los documentos adjuntos sean válidos. |
| ECU-17 | Generar número de radicado | Al finalizar el registro de una solicitud, el sistema genera automáticamente un identificador único para facilitar el seguimiento del proceso. |
| ECU-18 | Registrar trazabilidad de acciones | El sistema almacena automáticamente cada acción realizada sobre una solicitud, incluyendo fecha, hora, usuario responsable y cambio ejecutado. |
