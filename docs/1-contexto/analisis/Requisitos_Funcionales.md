# Requisitos Funcionales

| Código | Requerimiento Funcional |
|---|---|
| RF-01 | El sistema debe permitir a los empleados registrar solicitudes de permisos laborales, vacaciones, incapacidades médicas y certificaciones laborales mediante una plataforma centralizada. |
| RF-02 | El sistema debe generar automáticamente una confirmación de registro que incluya número de radicado, fecha y hora de creación de la solicitud. |
| RF-03 | El sistema debe mostrar el estado actualizado de cada solicitud mediante estados definidos como “Pendiente”, “En revisión”, “Aprobada” o “Rechazada”. |
| RF-04 | El sistema debe asignar automáticamente o manualmente un responsable del área de Talento Humano para gestionar cada solicitud registrada. |
| RF-05 | El sistema debe permitir al área de Talento Humano aprobar, rechazar o dejar en revisión las solicitudes registradas. |
| RF-06 | El sistema debe almacenar el historial completo de acciones realizadas sobre cada solicitud para garantizar trazabilidad y seguimiento del proceso. |
| RF-07 | El sistema debe enviar notificaciones automáticas a los empleados cuando una solicitud sea registrada, actualizada, aprobada o rechazada. |
| RF-08 | El sistema debe permitir adjuntar documentos de soporte en solicitudes que requieran validación documental, como incapacidades médicas o certificaciones laborales. |
| RF-09 | El sistema debe permitir a los empleados consultar el estado y detalle de sus solicitudes en tiempo real. |
| RF-10 | El sistema debe permitir al área de Talento Humano consultar, filtrar y administrar solicitudes según tipo, estado, fecha o empleado. |
| RF-11 | El sistema debe mostrar el nombre o cargo del responsable encargado de revisar cada solicitud registrada. |
| RF-12 | El sistema debe mantener un identificador único o número de radicado para cada solicitud registrada en la plataforma. |
| RF-13 | El sistema debe generar reportes relacionados con solicitudes registradas, tiempos de respuesta y estados de aprobación o rechazo. |
| RF-14 | El sistema debe permitir el acceso desde navegadores web y dispositivos móviles con conexión a internet. |
| RF-15 | El sistema debe centralizar toda la información relacionada con solicitudes laborales en una única plataforma institucional. |
| RF-16 | El sistema debe validar que los campos obligatorios estén completos antes de permitir el envío de una solicitud. |
| RF-17 | El sistema debe restringir el acceso a la información según el rol y permisos asignados a cada usuario registrado. |
| RF-18 | El sistema debe permitir a los empleados consultar el historial de solicitudes realizadas anteriormente junto con sus respuestas y estados registrados. |
