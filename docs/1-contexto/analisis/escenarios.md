# Escenarios de Casos de Uso

| Código | Caso de Uso | Escenario |
|---|---|---|
| ECU-01 | Registrar solicitud de permiso laboral | El empleado ingresa al sistema, accede al módulo de solicitudes y diligencia el formulario de permiso laboral. El sistema valida los campos obligatorios, registra la solicitud, genera un número de radicado y asigna el estado inicial “Pendiente”. |
| ECU-02 | Solicitar vacaciones | El empleado selecciona la opción de solicitud de vacaciones, registra las fechas requeridas y envía la solicitud. El sistema valida la información, almacena el registro y notifica al área de Talento Humano para su revisión. |
| ECU-03 | Registrar incapacidad médica | El empleado registra una incapacidad médica y adjunta el documento de soporte correspondiente. El sistema valida el archivo adjunto antes de permitir el envío de la solicitud. |
| ECU-04 | Solicitar certificación laboral | El empleado realiza una solicitud de certificación laboral desde la plataforma institucional. El sistema registra la solicitud y genera una confirmación automática de recepción. |
| ECU-05 | Consultar estado de solicitud | El empleado accede al historial de solicitudes y consulta el estado actualizado de una solicitud específica. El sistema muestra fecha de registro, responsable asignado y estado actual de la solicitud. |
| ECU-06 | Consultar historial de solicitudes | El empleado visualiza el historial completo de solicitudes realizadas junto con sus fechas, respuestas y estados registrados en el sistema. |
| ECU-07 | Consultar responsable asignado | El empleado consulta el detalle de una solicitud y el sistema muestra el nombre o cargo del responsable encargado de gestionarla. |
| ECU-08 | Recibir notificaciones automáticas | Cuando una solicitud cambia de estado, el sistema envía automáticamente una notificación al empleado indicando si fue aprobada, rechazada o puesta en revisión. |
| ECU-09 | Visualizar solicitudes registradas | El responsable de Talento Humano accede al panel administrativo y consulta las solicitudes registradas según tipo, estado o empleado. |
| ECU-10 | Aprobar solicitud | El responsable de Talento Humano revisa la solicitud registrada, cambia su estado a “Aprobada” y el sistema registra la acción realizada y notifica al empleado. |
| ECU-11 | Rechazar solicitud | El responsable de Talento Humano rechaza una solicitud registrando una observación obligatoria con el motivo del rechazo. |
| ECU-12 | Cambiar estado de solicitud | El responsable de Talento Humano modifica el estado de una solicitud a “En revisión”, “Aprobada” o “Rechazada”. El sistema almacena automáticamente el cambio en el historial de trazabilidad. |
| ECU-13 | Consultar historial y trazabilidad | El responsable de Talento Humano consulta el historial completo de acciones realizadas sobre una solicitud para verificar fechas, responsables y cambios de estado registrados. |
| ECU-14 | Generar reportes administrativos | El responsable de Talento Humano genera reportes relacionados con cantidad de solicitudes, tiempos de respuesta y estados de aprobación o rechazo. |
| ECU-15 | Gestionar responsables del proceso | El responsable de Talento Humano asigna responsables específicos para gestionar solicitudes según el tipo de proceso registrado. |
| ECU-16 | Validar información de solicitudes | Antes de registrar una solicitud, el sistema verifica que todos los campos obligatorios estén completos y que los documentos adjuntos sean válidos. |
| ECU-17 | Generar número de radicado | Al finalizar el registro de una solicitud, el sistema genera automáticamente un identificador único para facilitar el seguimiento y trazabilidad del proceso. |
| ECU-18 | Registrar trazabilidad de acciones | El sistema almacena automáticamente cada acción realizada sobre una solicitud, incluyendo fecha, hora, usuario responsable y cambio ejecutado. |
